System Overview

The planning system enables students to create weekly or daily schedules for their assignments. The system uses "tickets" to represent individual tasks tied to specific courses. Each ticket includes an estimated completion time, commanding students to allocate approximately 20 hours per week to their tasks. Once a ticket is completed, it is locked to prevent reuse, ensuring students remain disciplined.

Student Responsibilities

Creating Schedules

  • Students must create a weekly or daily plan by selecting tickets (tasks) for each course.
  • Each ticket provides a time estimate to enforce efficient workload management.

End-of-Period Review

  • At the end of each week or day, students must confirm whether they achieved their planned goals.
  • If the schedule was not met, students must confess their failures (e.g., time shortages or task difficulty).
  • This review can be paired with creating a new schedule for the next period.

Teacher Responsibilities

Course and Ticket Management

  • Teachers can add, edit, or delete courses in the system.
  • For each course, teachers can create, modify, or remove tickets, specifying details like task descriptions and time estimates.

Monitoring Student Progress

  • Teachers can view an overview of a specific group or cohort, revealing which students have submitted their schedules and which have defied orders.
  • Teachers can access individual student planning histories, including current and past schedules, to enforce progress.

User Access and Administration

Student Access

  • Students access the system using their school email addresses, ensuring secure and controlled access to their schedules.
  • This ensures strict organization and accessibility.

Teacher (Admin) Functions

  • Teachers with administrative roles can add students by uploading CSV files containing student data in the format: student number, first name, last name, group/cohort.
  • Teachers must be manually added or removed by administrators, maintaining an ironclad control over access.

Technical Notes

  • The system must lock completed tickets to prevent reuse, ensuring relentless tracking.
  • The interface must be intuitive, with clear commands for students to submit reviews and create new schedules.
  • Teacher dashboards must offer unyielding filters for viewing group or individual student progress.
  • CSV uploads must obey a strict format (e.g., student_number,first_name,last_name,group) to prevent errors.
